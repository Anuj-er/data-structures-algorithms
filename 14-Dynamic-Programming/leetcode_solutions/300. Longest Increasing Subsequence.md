# Intuition
The goal is to find the length of the longest subsequence where numbers are strictly increasing. At every index `i`, we look back at all previous numbers `j`. If `nums[j] < nums[i]`, then the LIS ending at `i` could potentially be the LIS ending at `j` plus the current number.

# Approach
We use a 1D DP array to track lengths and a parent array to reconstruct the actual sequence.

## ðŸš€ Iterative DP with Reconstruction
**Why this approach?**  
This solution doesn't just return the length; it explicitly tracks how we got there using a parent array `p[]`. 

**Key Steps:**
1. **Initialize**: Every element is a subsequence of length 1.
2. **Inner Loop**: Compare the current element with all previous elements.
3. **Update**: `lis[i] = 1 + lis[prev]` if it leads to a longer chain.
4. **Reconstruct**: Use an `ArrayList` and backtrack through the `p[]` pointers.

```java []
class Solution {

    public int lengthOfLIS(int[] nums) {
        int n = nums.length;
        int lis[] = new int[n];
        int p[] = new int[n];
        int maxLen = 1;
        int maxInd = 0;
        for(int i=0;i<n;i++){
            lis[i] = 1;
            p[i] = i;
        }

        for(int i=1;i<n;i++){
            for(int prev = 0;prev<i;prev++){
                if(nums[prev] < nums[i]){
                    if(lis[i] < 1 + lis[prev]){
                        lis[i] = 1 + lis[prev];
                        p[i] = prev;
                    }
                }
            }
            if(lis[i] > maxLen){
                maxLen = lis[i];
                maxInd = i;
            }
        }

        ArrayList<Integer> list = new ArrayList<>();
        list.add(nums[maxInd]);
        while(maxInd != p[maxInd]){
            maxInd = p[maxInd];
            list.add(0, nums[maxInd]);
        }
        for(int i = 0;i<list.size();i++){
            System.out.print(list.get(i) + " ");
        }
        return maxLen;
    }
}
```

**ðŸŽ¯ Key Takeaways:**
- **Reconstruction Pattern**: Storing the predecessor index is a powerful technique in DP for visualizing the path.
- **Strictly Increasing**: The `nums[prev] < nums[i]` condition ensures the sequence never plateaus or decreases.
- **Scaling**: For very large arrays ($N > 2500$), you might need the $O(N \log N)$ approach using Binary Search.
